# Codex 参考型工作区重构设计

日期：2026-06-28

## 目标

在不增加后端能力、不改变现有业务行为的前提下，参考 Codex 桌面客户端的信息架构和交互层级，重构 Po Agent 的桌面工作区。

本次设计解决以下问题：

- 项目、会话、文件和配置入口缺少清晰层级。
- 左侧会话与文件树长期争夺空间。
- Chat、文件预览和配置功能使用不同的布局语言。
- 当前界面的视觉层级、密度和状态表达弱于 Codex 参考界面。

## 产品边界

- 产品仅支持桌面 Web，最低视口宽度为 1024px。
- 小于 1024px 时维持桌面布局并允许横向滚动，不提供移动端抽屉或移动端专用视图。
- 不新增搜索、自动化、设置中心、文件编辑、多文件标签页或“在编辑器中打开”。
- 不修改后端能力、HTTP 合同、SSE 生命周期或文件访问边界。
- 保留现有 Chat、Session、Model、Skills 和 File 功能。

## 视觉方向

采用“Codex 结构 + 现有主题”：

- 参考 Codex 的项目中心导航、中央主工作区、按需文件工作区和安静的信息层级。
- 保留 Po Agent 的松绿浅色主题与炭黑深色主题。
- 不复制 Codex 的蓝灰色品牌表达，也不引入当前项目没有的功能。

### 颜色规则

- 浅色主题继续使用 `canvas #f1f3f2`、`panel #f8f9f8`、`elevated #fbfcfb`、`text #151816`、`accent #17624b`。
- 深色主题继续使用 `canvas #121514`、`panel #181b1a`、`elevated #1f2421`、`text #eef1ed`、`accent #8bc7b0`。
- Accent 仅用于焦点、主操作、选择和实时状态。
- 普通面板保持无阴影；阴影仅用于 Popover、Tooltip 和必要 Dialog。
- 首次访问跟随系统主题；用户手动切换后继续使用现有本地持久化行为。

## 全局信息架构

工作区由三部分组成：

1. 左侧导航，可拖拽调宽。
2. 中央主工作区，在 Chat、Model Provider 和 Skills 之间切换。
3. 右侧文件工作区，默认关闭，由用户手动打开，可拖拽调宽。

### 左侧导航

从上到下固定为：

1. 新对话。
2. Model Provider / 模型供应商。
3. Skills / 技能。
4. 项目列表，以及各项目下嵌套的会话。
5. 底部主题与语言图标按钮。

项目是一级导航，会话属于项目。当前项目和当前会话使用背景、字重和结构边界共同表达选中状态。

“项目”分组标题右侧提供两个 Icon Button：

- 打开项目：复用当前项目选择能力，支持默认目录、最近目录和自定义绝对路径。已经出现于项目列表中的最近目录不重复展示在选择器中。
- 刷新：复用当前会话刷新能力，更新项目及其会话列表。

会话继续保留新建、选择、重命名、删除、Draft 和父子分支展示能力。新建会话的主入口统一为左侧顶部“新对话”。

底部主题与语言采用并列双图标工具栏：

- 图标反映当前状态。
- Hover 和键盘聚焦时通过 Tooltip 提供动作说明。
- 不增加“设置”入口。

### 中央主工作区切换

- 点击会话或新对话进入 Chat。
- 点击 Model Provider 进入全局模型供应商配置页面。
- 点击 Skills 进入当前项目的技能页面。
- Chat 始终保持挂载；进入配置页面时仅隐藏 Chat。
- 返回 Chat 后保留草稿、滚动位置、Minimap、流式任务和运行状态。
- 从配置页面选择项目或会话时直接进入对应 Chat。

## Chat 设计

Chat 的功能、状态机和现有交互保持不变，只重构视觉层级。

### 顶栏

Chat 顶栏只包含：

- 左侧栏展开/收起 Icon Button。
- 当前会话标题。
- 当前项目名称，作为次级信息。
- 右侧现有文件工作区 Icon Button。

顶栏不再放置 Model、Skills、主题或语言入口，也不增加文字版“文件”或“在编辑器中打开”。

Model Provider 页面顶栏显示对应本地化标题。Skills 页面顶栏显示对应本地化标题和当前项目名称。

### Message

- 用户消息使用轻量背景并靠右显示。
- 助手最终回复直接呈现，不套普通卡片。
- 保留复制、编辑、Fork 等现有消息操作。
- 保留 Markdown、代码块、图片和错误展示。
- 不恢复原始 Token 消耗或估算速度等非必要指标。

### 执行过程

- 连续 Thinking、工具调用和执行说明继续组成一个执行过程。
- 保留过程折叠、步骤展开、工具状态、局部错误和最终回复分离逻辑。
- 工具摘要、状态和 Disclosure 控件继续使用稳定列，避免长命令挤压状态。

### Input

- 保留图片附件、模型选择、Thinking、Tools、Compact、Queue、Steer、Stop、发送快捷键和所有错误状态。
- 继续使用主操作与代理设置两行结构。
- 运行中控制保持可见，不因视觉重构改变优先级或忙碌保护。
- Compact 的可用性继续来自后端结果；禁用时保留原因 Tooltip。

### Minimap

- 保留显示条件、节点预览、Hover Tooltip、点击和拖拽导航。
- Minimap 固定在 Chat 右缘，不进入文件工作区。
- 短会话继续自动隐藏 Minimap。

### Welcome

- 保留现有说明和四个仅填充输入框的 Starter Prompt。
- Starter Prompt 不直接提交消息。
- Welcome 视觉可以按新的密度、间距和颜色体系调整。

## 文件工作区

- 默认关闭，仅由用户点击现有文件工作区 Icon Button 或明确选择文件后打开。
- 切换项目、进入会话和工具调用不会自动打开文件工作区。
- 文件工作区打开后，Chat 缩窄，文件区域获得更大的阅读宽度。
- 关闭后 Chat 恢复完整宽度。
- 外层文件工作区继续支持拖拽调宽，并恢复上次宽度。

### 内容结构

- 文件预览位于文件工作区左侧。
- 文件树位于文件工作区右侧。
- 文件树不提供搜索或筛选。
- 保留目录展开、目录加载、文件打开、刷新和 `@` 引用功能。
- 只保留当前已有的单文件只读预览。
- 不增加语法编辑器、标签页、文件修改或外部编辑器入口。
- 文件读取 Loading、Empty 和 Error 状态在文件工作区内展示。

进入 Model Provider 或 Skills 时暂时隐藏文件工作区，但保留其打开状态、当前文件和宽度；返回 Chat 时恢复。

## Model Provider 页面

- 左侧导航英文名称为 `Model Provider`，中文名称为 `模型供应商`。
- 页面属于全局配置，不依赖当前项目。
- 将现有 ModelsConfigDialog 的最外层 Dialog 壳层替换为中央主工作区页面。
- 复用 `useModelsConfig`、供应商导航、详情组件、发现、测试、保存和凭据操作。
- API Key、OAuth、删除确认等细节操作继续使用必要 Dialog。
- 页面有未保存修改时，切换会话、项目、Skills 或其他主视图必须触发放弃修改确认。
- 保存成功后继续更新 Chat 的模型列表。

## Skills 页面

- 左侧导航英文名称为 `Skills`，中文名称为 `技能`。
- 页面绑定当前项目目录。
- 将现有 SkillsConfigDialog 的最外层 Dialog 壳层替换为中央主工作区页面。
- 复用 `useSkillsConfig`、SkillList、SkillDetail、AddSkillPanel、刷新、安装、启用状态和诊断展示。
- 未选择项目时 Skills 导航禁用，并通过 Tooltip 明确说明需要先选择项目。
- 安装和状态修改继续保留 Busy Guard、错误提示和重试入口。

## 前端模块边界

遵守 `feature -> layout` 禁止反向依赖的现有规则：

- `src/layouts/agent-workspace` 负责应用壳层、活动视图和跨 Feature 状态协调。
- `src/features/chat` 继续独立拥有 Chat 行为。
- `src/features/session-sidebar` 只拥有项目、会话和会话操作，不再拥有文件树。
- 文件树迁入 `src/features/file-panel`，与单文件预览组成 File Workspace。
- `src/features/models-config` 和 `src/features/skills-config` 分别提供可嵌入中央区域的页面内容。
- Layout 通过 Props 和 Callback 协调会话选择、项目选择、文件打开和配置保存。
- 不让一个 Feature 直接导入另一个 Feature。
- 不新增后端层或客户端 API 合同。

## 状态与导航规则

- 活动视图为 `chat | model-provider | skills`。
- Chat Controller 生命周期不依赖活动视图。
- 新对话、项目选择和会话选择会切换至 Chat。
- 文件工作区只在 Chat 视图呈现。
- 切换配置视图不会停止正在运行的 Agent。
- Model Provider 的未保存保护优先于视图切换。
- Skills 的项目上下文始终来自活动项目，不从组件内部猜测。

## 错误与安全

- 保留当前模型凭据保护，响应、日志和 UI 状态不得暴露密钥。
- 保留模型测试与技能安装的副作用警告、Busy Guard 和错误状态。
- 文件访问继续限制在已注册 Workspace Root 内。
- 文件树迁移不改变文件 API 或路径验证。
- 可恢复错误在对应页面或步骤内展示，不升级为全局错误。
- 禁用控件必须通过 Tooltip 解释具体原因。
- Model Provider 的放弃修改、供应商删除、模型删除、API Key 删除和 OAuth 断开继续使用明确确认。

## 可访问性与国际化

- 所有 Icon Button 必须有本地化 `aria-label` 和 Tooltip。
- 保留键盘导航、Visible Focus 和语义 HTML。
- 浅色与深色主题达到 WCAG 2.1 AA 对比要求。
- 保留 `prefers-reduced-motion` 行为。
- 所有新增、删除和重命名文案同步更新英文、中文词典及其测试。
- 项目路径和技术值继续使用等宽字体；普通导航和表单使用 UI Sans。

## 实施顺序

1. 重构应用壳层、活动视图状态和左侧项目导航。
2. 将 Model Provider 与 Skills 的最外层 Dialog 转为中央页面。
3. 将文件树迁入右侧 File Workspace，并保留文件操作。
4. 在不改变行为的前提下重构 Message、Input、Minimap 和 Welcome 视觉。
5. 清理移动端分支，统一桌面宽度、主题 Token 和中英文文案。
6. 完成测试、生产构建和浏览器验收。

## 测试与验收

### 自动化测试

- 保留并通过现有 Chat、Session、Model、Skills、File 测试。
- 更新已改变的视觉合同测试，删除只验证旧位置或旧 Dialog 壳层的断言。
- 增加项目嵌套会话和左侧导航顺序测试。
- 增加打开项目、默认目录、自定义路径、刷新和最近项目去重测试。
- 增加 Chat 保持挂载、视图切换和运行状态保留测试。
- 增加 Model Provider 未保存修改导航保护测试。
- 增加 Skills 无项目禁用原因测试。
- 增加文件工作区手动开关、隐藏恢复、宽度恢复和文件树操作测试。
- 增加主题与语言底部图标位置和本地化标签测试。
- 增加 1024px 最低桌面布局合同测试。

### 浏览器验收

验证 1024px、1440px 和 1920px 宽度：

- 浅色与深色主题。
- 英文与中文界面。
- 新对话、项目切换、会话切换和会话操作。
- Chat 消息、执行过程、Input 全状态和 Minimap。
- 文件工作区开关、调宽、文件树、文件读取和 `@` 引用。
- Model Provider 加载、编辑、测试、保存和放弃修改确认。
- Skills 加载、刷新、安装、启用状态和诊断。
- 键盘焦点、Tooltip、错误、Loading、Empty 和 Disabled 状态。

## 非目标

- 移动端和小屏专用界面。
- 新的项目搜索、会话搜索或文件搜索。
- Codex 插件、自动化、多 Agent 或 Worktree 功能。
- 文件编辑器、多标签页和外部编辑器集成。
- 后端接口、数据模型或 SSE 协议重构。
- 与本次工作区重构无关的代码整理。
